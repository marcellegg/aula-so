---

## 🖥️ Introdução aos Sistemas Operacionais

Um **Sistema Operacional (SO)** é o software responsável por gerenciar os recursos do computador e permitir a comunicação entre **hardware, aplicações e usuário**.

### Principais funções do Sistema Operacional
- Gerenciamento de processos
- Gerenciamento de memória
- Controle de dispositivos
- Execução de programas
- Segurança e estabilidade do sistema

---

# 🧠 Tipos de Sistemas Operacionais

## 🏢 Sistemas de Grande Porte (Mainframes)

São sistemas projetados para processar **grandes volumes de dados e muitas transações simultâneas**.

### Características
- Alta confiabilidade
- Grande capacidade de processamento
- Segurança e integridade de dados
- Suporte a múltiplos usuários

### Exemplos
- OS/360  
- OS/390  
- Linux em mainframes  
- Variantes do UNIX  

### Utilização
- Bancos
- Grandes empresas
- Sistemas de e-commerce

---

## 🌐 Sistemas Operacionais de Servidor

São sistemas voltados para **atender múltiplos usuários e serviços pela rede**.

### Funções principais
- Hospedagem de sites
- Servidores de arquivos
- Bancos de dados
- Autenticação de usuários

### Exemplos
- Linux
- Windows Server

Esses sistemas priorizam **estabilidade, escalabilidade e compartilhamento de recursos**.

---

## ⚙️ Sistemas de Multiprocessadores

São sistemas capazes de utilizar **vários processadores ou múltiplos núcleos de CPU simultaneamente**.

### Desafios
- Balanceamento de carga entre núcleos
- Sincronização de processos
- Coerência de cache

### Aplicações
- Servidores de alto desempenho
- Computação científica

---

## 💻 Sistemas de Computadores Pessoais

São sistemas utilizados em **desktops e notebooks**.

### Características
- Interface gráfica
- Suporte a múltiplos programas
- Compatibilidade com diversos softwares
- Recursos multimídia

### Exemplos
- Windows
- macOS
- Linux

---

## 📱 Sistemas Operacionais Portáteis

São sistemas desenvolvidos para **dispositivos móveis**, como smartphones e tablets.

### Características
- Gerenciamento de energia
- Integração com sensores (GPS, câmera, acelerômetro)
- Segurança baseada em permissões
- Distribuição de aplicativos por lojas oficiais

### Exemplos
- Android
- iOS

---

## 🔌 Sistemas Embarcados

São utilizados em **dispositivos dedicados**, geralmente com funções específicas.

### Exemplos de aplicação
- Micro-ondas
- Smart TVs
- Sistemas automotivos
- Equipamentos industriais

### Características
- Recursos limitados
- Software armazenado em ROM ou memória flash
- Usuário geralmente não altera o sistema

---

## 📡 Sistemas de Nós Sensores

São sistemas utilizados em **dispositivos pequenos conectados em rede**, geralmente alimentados por bateria.

### Aplicações
- Monitoramento ambiental
- Agricultura de precisão
- Vigilância

### Exemplos de sistemas
- TinyOS
- Contiki

### Características
- Baixo consumo de energia
- Comunicação sem fio
- Sistemas orientados a eventos

---

## ⏱️ Sistemas de Tempo Real

São sistemas onde o **tempo de resposta é crítico**.

### Tipos

**Hard Real-Time**
- Não pode perder prazos
- Exemplo: controle de voo

**Soft Real-Time**
- Pequenos atrasos são tolerados
- Exemplo: streaming de vídeo

---

## 💳 Sistemas de Cartões Inteligentes

Utilizados em **cartões com chip**, como cartões bancários ou de identificação.

### Características
- Recursos limitados
- Alto nível de segurança
- Uso de criptografia
- Execução de pequenos programas (applets)

---

# 🔧 Introdução ao Git

## O que é Git

O **Git** é um sistema de **controle de versão**, utilizado para acompanhar mudanças em arquivos e projetos.

### O que o Git permite
- Registrar versões de um projeto
- Restaurar versões anteriores
- Trabalhar em equipe
- Sincronizar projetos com repositórios online

---

## ⚙️ Instalação do Git

Passos básicos:

1. Baixar o instalador no site oficial
2. Executar o instalador
3. Seguir as etapas de instalação
4. Testar no terminal:


Se aparecer a versão instalada, o Git está funcionando corretamente.

---

## 📁 Criando um Repositório

Passos básicos:

1. Criar uma pasta do projeto
2. Abrir a pasta no VS Code
3. Criar um arquivo `.md`
4. Inicializar o repositório Git
5. Fazer um commit

---

## ☁️ Publicando no GitHub

Etapas:

1. Publicar a branch
2. Fazer login no GitHub
3. Escolher repositório público ou privado
4. Enviar os arquivos para o repositório remoto

---

# ✅ Boas Práticas com Git

- Fazer **commits pequenos e frequentes**
- Escrever **mensagens de commit claras**
- Utilizar **branches para novas funcionalidades**
- Realizar **testes antes de fazer merge**
